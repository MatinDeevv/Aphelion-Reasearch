1. Executive Summary
APHELION is a fully autonomous, self-evolving algorithmic trading system built exclusively for XAU/USD (Gold) intraday trading. It operates on a Google Cloud Platform virtual machine equipped with an NVIDIA L4 GPU, 128GB RAM, and 32 CPU cores, running 24 hours a day, 7 days a week.
The system consists of 20 primary modules containing over 100 sub-components, governed by a 5-tier hierarchy with weighted voting. It trades during active market sessions (London and New York), evolves its own strategies during idle hours, and trains on extreme simulated market conditions via the NEMESIS simulation engine.
The end goal is twofold: first, to compound a $10,000 starting account at an estimated 1.8-2.8% per day using institutional-grade signals and dynamic leverage up to 100x under strict risk controls; second, to generate 50,000+ labeled trading decisions that will be used to fine-tune GPT-OSS-120B — producing APHELION-120B, a private large language model that replaces the entire 20-module stack with a single fine-tuned neural network.

1.2 System Philosophy
APHELION is built on three principles:
Survive first, profit second. SENTINEL's risk rules are hardcoded and cannot be overridden by any component, vote, or evolution cycle. No trade is worth blowing the account.
Complexity earns its place. Every component must demonstrably improve system performance. Components that do not contribute measurable edge are retired by CIPHER.
The system improves itself. PROMETHEUS continuously evolves better trading strategies using genetic algorithms and Bayesian optimization. The system running tonight is better than the system that ran yesterday.

4. The 20 Modules — Complete Specification
4.1 HYDRA — Neural Intelligence Core
Absorbs: HYDRA-X, KRONOS, ECHO, ECHO-X
Purpose
The primary prediction engine. HYDRA is a neural ensemble of four specialized models unified by a dynamic attention gate. Each sub-model specializes in a different aspect of market behavior.
Sub-Components
SUB-COMPONENT
DESCRIPTION
HYDRA-TFT
Temporal Fusion Transformer. Multi-horizon prediction with variable selection networks. Outputs P(LONG)/P(SHORT)/P(FLAT) with uncertainty bounds across 5m, 15m, 1h horizons.
HYDRA-LSTM
Bidirectional LSTM with self-attention. Captures sequential momentum and regime persistence. Maintains separate hidden states per timeframe.
HYDRA-CNN
Convolutional ResNet applied to 64-bar OHLCV images (4-channel input). Detects geometric candle patterns and structural formations. Outputs 128-dim regime embedding.
HYDRA-MOE
Mixture of Experts with 4 specialist models: TREND, RANGE, VOL_EXPANSION, NEWS_REACTION. Gating transformer routes each input to the appropriate expert.
HYDRA-GATE
Dynamic attention gate. Weights all four model outputs based on real-time regime classification. Replaces static ensemble averaging.
KRONOS
TimesFM foundation model fine-tuned on 10 years of XAU/USD tick data. Provides probabilistic multi-horizon forecasts (P10/P50/P90) as a fifth ensemble input.
ECHO
FAISS vector database of 50,000 historical 500-bar windows from 2000-2019. KNN retrieval finds the K most similar historical market situations and reports what happened next.
ECHO-X
Separate FAISS database covering 2020-present. Stores post-COVID, post-rate-hike, and crypto-correlation-era analogs which behave distinctly from pre-2020 patterns.

60+ engineered features, raw OHLCV, tick microstructure data

HYDRA_SIGNAL (LONG/SHORT/FLAT), HYDRA_CONFIDENCE (0-1), HYDRA_UNCERTAINTY, regime classification

Absorbs: FORGE, CIPHER, MERIDIAN, PRISM

Sub-Components
SUB-COMPONENT
DESCRIPTION
FORGE-MAML
Model-Agnostic Meta-Learning. Performs 5 gradient steps on the most recent 100 bars during live trading. Adapts HYDRA weights to current regime without full retraining.
FORGE-REGIME
Rapid regime shift detector. Uses change-point analysis on rolling VPIN and entropy to detect regime transitions within 30-90 minutes.

Rolling SHAP importance tracker. Computes 30-day vs 90-day SHAP ratio for every feature. Detects alpha decay before it impacts performance.

MERIDIAN-WEIGHTS
Dynamic timeframe weight vector [w_1m, w_5m, w_15m, w_1h]. Replaces static MTF alignment score with causality-derived weights updated every 50 bars.
PRISM
Multi-timeframe signal decomposer. Breaks every signal into its per-timeframe contribution. Identifies which timeframe is generating the edge on each trade.

4.3 PROMETHEUS — Evolution & Genetic Optimization Engine
Absorbs: PROMETHEUS, ZEUS, MATRIX
Purpose
The self-improvement engine. PROMETHEUS continuously evolves better trading strategies using three complementary evolutionary methods running in parallel.
Sub-Components
SUB-COMPONENT
DESCRIPTION
PROMETHEUS-NEAT
Neuroevolution of Augmenting Topologies. Evolves both the weights AND topology of trading neural networks. Population size 48. Bottom 25% killed each generation.
PROMETHEUS-PBT
Population-Based Training (DeepMind method). N models train simultaneously. Bottom 20% copy top 20% weights then mutate hyperparameters.
PROMETHEUS-BAYES
Bayesian Optuna TPE hyperparameter search. Prunes underperforming trials early using Hyperband. Covers: learning rates, feature sets, risk parameters, architecture choices.
PROMETHEUS-FITNESS
Multi-objective fitness function: fitness = Sharpe*0.30 + DSR*0.20 + ProfitFactor*0.20 + (1-MDD)*0.15 + RegimeConsistency*0.10 + WinRateConsistency*0.05
PROMETHEUS-DEPLOY
Genome promotion pipeline. Walk-forward validation → Monte Carlo 500 paths → ZEUS stress test → paper trading → live deployment. Full pipeline before any genome touches real money.
ZEUS-STRESS
Pre-deployment stress tester. Every genome must survive: flash crash simulation, 10x spread blowout, 500ms latency spikes, data gap injection, and LEVIATHAN adversarial attacks.
ZEUS-GAN
GAN-generated synthetic regime generator. Creates never-seen market conditions to test for overfitting. Genomes that fail GAN regimes are rejected.
MATRIX
Parallel paper trading simulation. Runs every rejected signal in a virtual account. Tracks what WOULD have happened. Feeds missed opportunity data back to PROMETHEUS.

DETAIL

Historical tick data (10 years), feature engine output, SENTINEL risk parameters

6.3 Kelly Criterion Position Sizing
TITAN applies the Kelly criterion to every approved trade:
f = (b*p - q) / b
where: b = R:R ratio, p = HYDRA win probability, q = 1 - p
Maximum f applied: 2% of account equity per trade
Fractional Kelly applied: f_actual = f * 0.25 (quarter-Kelly for safety)

12. Glossary of Terms
TERM
DEFINITION
Alpha
Trading edge above a random benchmark. The reason a strategy makes money.
Cointegration
A long-run equilibrium relationship between two price series that tends to revert to a mean spread.
CUDA
NVIDIA's GPU parallel computing platform. Required for all PyTorch neural network training.
Deflated Sharpe Ratio (DSR)
Sharpe ratio adjusted for multiple testing, non-normality, and autocorrelation. More conservative than raw Sharpe.
Genome
A complete specification of a trading strategy including model architecture, hyperparameters, and feature selection. Evolved by PROMETHEUS.
GGUF
File format for quantized LLM weights compatible with llama.cpp. Allows large models to run on consumer hardware.
Granger Causality
A statistical test for whether past values of variable X improve the prediction of variable Y beyond Y's own history.
Harmony Format
OpenAI's instruction format for GPT-OSS models. Required for correct model behavior.
Hawkes Process
A self-exciting point process where each event increases the probability of future events. Used to model clustered order arrivals.
Kelly Criterion
Mathematical formula for optimal bet sizing: f = (bp - q) / b. Maximizes long-run capital growth.
MAML
Model-Agnostic Meta-Learning. Trains a model to adapt quickly to new tasks with few gradient steps.
MoE
Mixture of Experts. Neural architecture where different sub-networks specialize in different input types.
Monte Carlo Simulation
Running thousands of random simulations to estimate probability distributions of outcomes.
NEAT
NeuroEvolution of Augmenting Topologies. Genetic algorithm that evolves both neural network weights and structure.
OFI
Order Flow Imbalance. Net signed volume across bid and ask sides. Directional indicator of aggressive order flow.
Overfitting
When a model learns historical noise rather than genuine signal. Performs well in-sample but poorly out-of-sample.
PBT
Population-Based Training (DeepMind). Multiple models train in parallel; underperformers copy and mutate top performers.
SHAP
SHapley Additive exPlanations. Game-theoretic method for attributing model predictions to individual features.
TFT
Temporal Fusion Transformer. Transformer architecture designed for multi-horizon time series forecasting with interpretability.
VPIN
Volume-Synchronized Probability of Informed Trading. Measures order flow toxicity. High values indicate informed traders active.
Walk-Forward Validation
Train on rolling window of past data, test on next unseen period. Most realistic backtest methodology.
XGBoost
Extreme Gradient Boosting. GPU-accelerated gradient boosted tree ensemble. Fast, interpretable, strong baseline for tabular data.
